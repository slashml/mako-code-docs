# API Endpoints

This page documents all available endpoints in the Mako Code API.

## Authentication

All API requests require authentication using an API key.

```bash
curl -H "Authorization: Bearer YOUR_API_KEY" https://api.mako-code.org/v1/endpoint
```

## Base URL

```
https://api.mako-code.org/v1
```

## Endpoints

### Users

#### Get User

```
GET /users/:id
```

Retrieves information about a specific user.

**Parameters:**

| Parameter | Type | Description |
|-----------|------|-------------|
| id | string | The unique identifier of the user |

**Response:**

```json
{
  "id": "user_123",
  "name": "John Doe",
  "email": "john@example.com",
  "created_at": "2023-01-01T00:00:00Z"
}
```

#### Create User

```
POST /users
```

Creates a new user.

**Request Body:**

```json
{
  "name": "Jane Doe",
  "email": "jane@example.com",
  "password": "secure_password"
}
```

**Response:**

```json
{
  "id": "user_124",
  "name": "Jane Doe",
  "email": "jane@example.com",
  "created_at": "2023-05-01T00:00:00Z"
}
```

### Projects

#### List Projects

```
GET /projects
```

Retrieves a list of projects.

**Query Parameters:**

| Parameter | Type | Description |
|-----------|------|-------------|
| limit | integer | Maximum number of projects to return (default: 10) |
| offset | integer | Number of projects to skip (default: 0) |

**Response:**

```json
{
  "projects": [
    {
      "id": "proj_123",
      "name": "Sample Project",
      "description": "A sample project",
      "created_at": "2023-01-01T00:00:00Z"
    },
    {
      "id": "proj_124",
      "name": "Another Project",
      "description": "Another sample project",
      "created_at": "2023-02-01T00:00:00Z"
    }
  ],
  "total": 2
}
```

#### Get Project

```
GET /projects/:id
```

Retrieves information about a specific project.

**Parameters:**

| Parameter | Type | Description |
|-----------|------|-------------|
| id | string | The unique identifier of the project |

**Response:**

```json
{
  "id": "proj_123",
  "name": "Sample Project",
  "description": "A sample project",
  "created_at": "2023-01-01T00:00:00Z",
  "owner_id": "user_123"
}
```

## Error Handling

The API uses conventional HTTP response codes to indicate the success or failure of an API request.

| Code | Description |
|------|-------------|
| 200 | OK - The request was successful |
| 400 | Bad Request - The request was invalid |
| 401 | Unauthorized - Authentication failed |
| 403 | Forbidden - The request is not allowed |
| 404 | Not Found - The requested resource does not exist |
| 429 | Too Many Requests - Rate limit exceeded |
| 500 | Server Error - Something went wrong on our end |

## Rate Limiting

The API implements rate limiting to prevent abuse. Current limits are:

- 100 requests per minute per API key
- 5,000 requests per day per API key

Rate limit headers are included in all API responses:

```
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1620000000
```
